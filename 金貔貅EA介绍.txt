# 金貔貅 v1.02 — 智能趋势网格交易系统

## 产品概述

**金貔貅**是一款专为黄金(XAU/USD)设计的MetaTrader 5智能交易系统，融合了**SMC智能资金概念**、**多周期趋势过滤**和**动态网格加仓**三大核心技术，实现"顺势建仓、逆势加仓、动态止盈"的全自动化交易闭环。

系统采用四层信号过滤体系（H4大周期方向 → H1中周期确认 → M15小周期精确 → CCI动量否决），确保每一次入场都在大势方向上具备多重确认。配合智能马丁网格加仓策略，在行情回撤时有序加仓摊平成本，在趋势恢复时通过动态止盈实现快速收割。

---

## 核心优势

### 1. SMC智能资金多维度信号系统
- **大周期(4H)**：Imbalance失衡检测 + Supply/Demand供需区识别
- **中周期(1H)**：Order Block订单块 + FVG公允价值缺口
- **小周期(15M)**：Liquidity Void流动性空白 + Breaker破坏块
- **动量过滤(H1)**：CCI极端否决机制，避免追涨杀跌

### 2. 动态止盈 + 追踪止损双保险
- **动态TP**：止盈金额随持仓层数自动递增，层数越多目标越高
- **追踪止损**：浮盈达到门槛后启动追踪，锁定大部分利润
- 两套机制独立运行，谁先触发谁先平仓

### 3. 五层风控安全网
- 日亏损百分比限制 → 点差异常过滤 → 快速亏损熔断 → 硬止损强平 → 对冲锁仓保护
- 从常规风控到极端行情全覆盖

### 4. 实时诊断面板
- 一目了然的可视化状态面板，实时显示评分、层数、浮盈、TP进度
- 一键操作：平多/平空/平盈利/平亏损/全平仓/暂停交易
- 历史交易明细面板：最近15天每日盈亏统计

### 5. 离线授权保护
- 无需联网验证，授权码绑定账号+有效期
- 安全可靠，不受网络波动影响

## 策略运行逻辑

```
┌─────────────────────────────────────────────────────────┐
│  ① 信号生成（多周期SMC + EMA综合评分）                    │
│     H4趋势方向 → H1结构确认 → M15精确时机 → CCI否决      │
├─────────────────────────────────────────────────────────┤
│  ② 首单入场（满足评分阈值 + 通过风控检查）                │
│     综合评分 ≥ 30分 → 方向一致 → 开仓首层                │
├─────────────────────────────────────────────────────────┤
│  ③ 网格加仓（行情逆向时有序加仓）                         │
│     间距递增（100+20×层数 点）→ 手数递增（×1.2倍/层）     │
├─────────────────────────────────────────────────────────┤
│  ④ 动态止盈（行情回归时快速收割）                         │
│     TP = 8 + (层数-1)×8 美分 → 追踪启动(TP×60%) → 平仓  │
├─────────────────────────────────────────────────────────┤
│  ⑤ 风控保护（极端行情安全网）                             │
│     日亏限制 → 硬止损 → 快速熔断 → 对冲锁仓              │
└─────────────────────────────────────────────────────────┘
```

## 参数设置指南

### 基础设置

| 参数 | 默认值 | 建议 |
|------|--------|------|
| 策略预设名称 | CENT_MARTINGALE | 自定义标识，便于多图表区分 |
| Magic Number | 26042702 | **每个图表必须唯一**，避免冲突 |
| 授权码 | (空) | 联系管理员获取，绑定交易账号 |

> **注意**：Magic Number必须为纯数字，同一账号不同EA必须使用不同Magic。

### 入场信号设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 入场模式 | 综合评分 | 推荐使用综合评分模式（EMA+SMC双重确认） |
| 综合评分阈值 | 30 | 越高越保守（建议25-45） |
| EMA快线周期 | 8 | 适合M5短周期捕捉 |
| EMA慢线周期 | 21 | 与快线配合形成金/死叉 |
| H4过滤 | 开启 | **强烈建议开启**，避免逆势交易 |
| CCI极端过滤 | 开启 | 避免极端行情入场 |

> **建议**：新手保持默认设置。如果发现入场过少，可将阈值降至25；入场过于频繁则提高至40-50。

### 马丁加仓设置

| 参数 | 默认值 | 注意事项 |
|------|--------|---------|
| 首单手数 | 0.01 | 根据账户资金调整 |
| 手数乘数 | 1.20 | 每层递增20%，越大风险越高 |
| 单层最大手数 | 1.50 | 防止单层手数失控 |
| 最大层数 | 50 | 根据资金量设定安全上限 |
| 基础间距 | 100点 | 黄金建议80-150点 |
| 每层增加间距 | 20点 | 深层间距更宽，降低集中风险 |
| 总手数上限 | 2.0手 | **关键安全参数**，必须设置 |

> **重要**：总手数上限是最后一道防线。美分账户50000美分建议总手数≤2.0手；资金越少，上限越低。

### 止盈止损设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 基础止盈(TP) | 8美分 | 1层目标8美分 |
| 每层TP增量 | 8美分 | 5层目标=8+(5-1)×8=40美分 |
| 追踪回撤比例 | 50% | 浮盈回撤50%时平仓 |
| 追踪启动门槛 | 60%(占TP) | 浮盈达到TP的60%时启动追踪 |
| 硬止损 | 0(关闭) | 设置后篮子亏损超此值强平 |
| 平仓冷却 | 45秒 | 防止频繁开平仓 |

> **提示**：动态TP公式 = 基础TP + (层数-1) × 每层增量。层数越多目标越高，保证深层有足够利润空间。

### 风控设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 日最大亏损 | 8% | 达到后当日停止交易 |
| 最大点差 | 450点 | 超过不开新仓 |
| 定时休市 | 20:00-23:00(北京) | 重要数据发布时段 |
| 快速亏损熔断 | 关闭 | 可选启用，极端行情保护 |
| 对冲保护 | 关闭 | 可选启用，深度浮亏时锁仓 |

> **注意**：
> - 定时休市期间会**自动平仓所有持仓**，请根据实际需要调整时段
> - 起止小时设为相同值(如都为0)则关闭此功能
> - 对冲支持两种触发模式：权益百分比 或 绝对金额

### 面板设置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 显示面板 | 开启 | 关闭可略微提升性能 |
| 面板宽度 | 680像素 | 根据屏幕分辨率调整 |
| 面板高度 | 380像素 | — |
| 刷新间隔 | 2秒 | 越短越实时，但更耗资源 |
| 历史天数 | 15天 | 显示最近N天交易统计 |

## 面板操作说明

### 主面板信息区
- **篮子浮盈**：当前所有持仓的实时盈亏
- **持仓手数**：当前总手数 和 层数/最大层数
- **TP进度**：距离止盈目标的完成百分比
- **SMC评分**：大/中/小三个周期的实时评分
- **综合得分**：多头和空头各自的总分

### 操作按钮
| 按钮 | 功能 |
|------|------|
| 平多单 | 平仓所有多头持仓 |
| 平空单 | 平仓所有空头持仓 |
| 平盈利 | 只平仓盈利的单子 |
| 平亏损 | 只平仓亏损的单子 |
| 全平仓 | 平仓所有持仓 |
| 暂停交易 | 暂停/恢复自动开仓 |
| 历史明细 | 打开/关闭右上角历史面板 |
| 隐藏面板 | 隐藏整个状态面板 |

## 使用注意事项

### 部署前检查
1. ✅ 确认账户类型为**对冲账户**(Hedging)
2. ✅ EA属性中勾选**"允许算法交易"**
3. ✅ 填入有效的**授权码**
4. ✅ 图表周期设为**M15**
5. ✅ Magic Number不与其他EA重复

### 资金管理建议
- 美分账户50000美分：首单0.01手，总上限2.0手，最大50层
- 美分账户100000美分：首单0.02手，总上限4.0手
- **切勿满仓操作**，建议实际使用资金不超过账户的50%

### 常见问题

**Q: EA加载后显示"授权验证失败"**
A: 检查授权码是否正确，是否绑定了当前登录的账号，是否在有效期内。

**Q: 面板显示"未建仓"但看起来条件满足**
A: 查看面板底部的具体原因提示（综合评分不足、H4过滤未通过、点差过大、冷却中等）。

**Q: 定时休市期间自动平仓了**
A: 这是设计行为。如不需要，将起止小时都设为0即可关闭。

**Q: 想在多个品种上运行**
A: 每个品种开一个M15图表，分别加载EA，**每个图表使用不同的Magic Number**。
